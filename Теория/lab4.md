1.	Проблема распределения криптографических ключей
это фундаментальная задача обеспечения безопасной передачи симметричных ключей между участниками связи без риска их перехвата.
---
2.1. Цели протоколов (проверка активности, аутентификация сторон, распределения ключа, подтверждение ключа, проверка новизны ключа). 
- проверка активности: Подтверждть что вторая сторона активна в данный момент
- аутентификация сторон: Подтверждение истинности личности участника
- Распределение ключа: Передача или создание общего секретного ключа между сторонами для последующего шифрования.
- Подтверждение ключа: Один участник получает доказательство того, что другой участник действительно владеет тем же самым секретным ключом.
- Проверка новизны ключа: Гарантия того, что ключ — новый и не был скомпрометирован в предыдущих сессиях.

2.2. С помощью каких механизмов достигаются данные цели.
- С помощью каких механизмов достигаются данные цели
- Асимметричное шифрование (RSA, ECC): Для обмена ключами и подписей.
- Симметричное шифрование (AES): Для быстрой передачи данных.
- Хэш-функции и MAC: Для проверки целостности.
---
3. Атаки на криптографические протоколы
- Атаки на криптографические протоколы — это методы компрометации защищенного обмена данными, нацеленные на использование уязвимостей в реализации, логике протокола или слабых алгоритмах, а не только на перебор ключей
- Человек посередине (Man-in-the-Middle, MITM): Злоумышленник встраивается в канал связи между участниками, перехватывая и модифицируя сообщения, оставаясь незамеченным.
- Атаки по времени (Timing Attack): Анализ времени, затрачиваемого устройством на выполнение криптографических операций, для определения секретного ключа.
